    Google Analytics 4 & Sentry Setup Guide

    Complete guide for setting up analytics and error tracking

    ðŸ“Š Google Analytics 4 Setup

    Prerequisites

    • Google account
    • Admin access to LocalBridge
    • Google Analytics property (or ability to create one)

    Step 1: Create Google Analytics 4 Property

    1. Go to Google Analytics
    2. Visit: https://analytics.google.com

    3. Sign in with your Google account

    4. Create Property

    5. Click Admin (bottom left)
    6. Click Create Property
    7. Enter property details:
      • Property name: LocalBridge
      • Reporting time zone: Your timezone
      • Currency: Your currency

    8. Click Next

    9. Business Information

    10. Industry category: Technology
    11. Business size: Select appropriate size

    12. Click Next

    13. Business Objectives

    14. Select: Examine user behavior

    15. Click Create

    16. Accept Terms

    17. Review and accept terms of service
    18. Click I Accept

    Step 2: Set Up Data Stream

    1. Choose Platform

    2. Select Web

    3. Configure Stream

    4. Website URL: https://your-domain.com
    5. Stream name: LocalBridge Web

    6. Click Create stream

    7. Get Measurement ID

    8. Copy the Measurement ID (format: G-XXXXXXXXXX)
    9. Keep this page open for later

    1. In Data Stream Settings
    2. Click Enhanced measurement

    ðŸš¨ Sentry Error Tracking Setup

    Prerequisites

    • Sentry account (free tier available)
    • Admin access to LocalBridge

    Step 1: Create Sentry Project

    1. Sign Up / Log In
    2. Visit: https://sentry.io

    3. Create account or log in

    4. Create Project

    5. Click Projects â†’ Create Project
    6. Platform: React
    7. Alert frequency: Alert me on every new issue
    8. Project name: localbridge-frontend
    9. Team: Select or create team

    10. Click Create Project

    11. Get DSN

    12. Copy the DSN (Data Source Name)

    13. Format: https://xxxxx@xxxxx.ingest.sentry.io/xxxxx

    14. Create Backend Project

    15. Repeat for backend
    16. Platform: Node.js
    17. Project name: localbridge-backend

    Step 2: Install Sentry SDK

    # Frontend
npm install @sentry/react

# Backend
npm install @sentry/node @sentry/profiling-node

    Step 3: Configure Frontend

    1. Update main.tsx
    // src/main.tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import * as Sentry from '@sentry/react';
import App from './App';
import './index.css';

// Initialize Sentry
if (import.meta.env.PROD && import.meta.env.VITE_SENTRY_DSN) {
  Sentry.init({
    dsn: import.meta.env.VITE_SENTRY_DSN,
    integrations: [
      new Sentry.BrowserTracing({
        // Set tracePropagationTargets to control distributed tracing
        tracePropagationTargets: [
          'localhost',
          /^https:\/\/yourserver\.io\/api/,
        ],
      }),
      new Sentry.Replay({
        maskAllText: true,
        blockAllMedia: true,
      }),
    ],

    // Performance Monitoring
    tracesSampleRate: 1.0, // Capture 100% of transactions

    // Session Replay
    replaysSessionSampleRate: 0.1, // 10% of sessions
    replaysOnErrorSampleRate: 1.0, // 100% of sessions with errors

    // Environment
    environment: import.meta.env.MODE,

    // Release tracking
    release: import.meta.env.VITE_APP_VERSION || '1.0.0',

    // Before send hook (filter sensitive data)
    beforeSend(event, hint) {
      // Don't send events in development
      if (import.meta.env.DEV) {
        return null;
      }

      // Filter sensitive data
      if (event.request) {
        delete event.request.cookies;
        delete event.request.headers;
      }

      return event;
    },
  });
}

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
    1. Update Environment Variables
    # .env.production
VITE_SENTRY_DSN=https://xxxxx@xxxxx.ingest.sentry.io/xxxxx
VITE_ENABLE_ERROR_REPORTING=true
VITE_APP_VERSION=1.0.0

    Step 4: Configure Backend

    1. Update server/index.ts
    // server/index.ts
import express from 'express';
import * as Sentry from '@sentry/node';
import { ProfilingIntegration } from '@sentry/profiling-node';

const app = express();

// Initialize Sentry (before any other middleware)
if (process.env.NODE_ENV === 'production' && process.env.SENTRY_DSN) {
  Sentry.init({
    dsn: process.env.SENTRY_DSN,
    integrations: [
      new Sentry.Integrations.Http({ tracing: true }),
      new Sentry.Integrations.Express({ app }),
      new ProfilingIntegration(),
    ],
    tracesSampleRate: 1.0,
    profilesSampleRate: 1.0,
    environment: process.env.NODE_ENV,
  });

  // RequestHandler creates a separate execution context
  app.use(Sentry.Handlers.requestHandler());

  // TracingHandler creates a trace for every incoming request
  app.use(Sentry.Handlers.tracingHandler());
}

// ... your routes ...

// Error handler (must be before any other error middleware)
if (process.env.NODE_ENV === 'production' && process.env.SENTRY_DSN) {
  app.use(Sentry.Handlers.errorHandler());
}

// ... rest of your app ...
    1. Update Environment Variables
    # .env.production (backend)
SENTRY_DSN=https://xxxxx@xxxxx.ingest.sentry.io/xxxxx
NODE_ENV=production

    Step 7: Set Up Source Maps

    1. Install Sentry CLI
    npm install --save-dev @sentry/vite-plugin
    1. Update vite.config.ts
    import { sentryVitePlugin } from '@sentry/vite-plugin';

export default defineConfig({
  plugins: [
    react(),
    sentryVitePlugin({
      org: 'your-org',
      project: 'localbridge-frontend',
      authToken: process.env.SENTRY_AUTH_TOKEN,
    }),
  ],
  build: {
    sourcemap: true, // Enable source maps
  },
});
    1. Create Sentry Auth Token
    2. Go to Sentry â†’ Settings â†’ Auth Tokens
    3. Create new token with project:releases scope
    4. Add to .env:
