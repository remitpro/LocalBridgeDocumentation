    Admin Login Guide

    üîê How to Create an Admin Account

    The LocalBridge application automatically makes the first user to register an admin.

    ‚úÖ Quick Solution

    If No Users Exist Yet (First Time Setup)

    1. Navigate to the registration page:
    2. Go to http://localhost:8081/register

    3. Or click "Get Started" from the login page

    4. Register with your details:

    5. Name: Your full name
    6. Email: Your email address

    7. Password: At least 8 characters

    8. Submit the form:

    9. You will automatically be assigned the admin role
    10. You'll be logged in immediately

    If Users Already Exist

    If you've already registered users and need admin access, you have two options:

    Option 1: Reset the Database (Development Only)

    # WARNING: This deletes all data!
npm run db:reset
npm run db:migrate

# Then register as the first user

    Option 2: Manually Update an Existing User to Admin

    Run this SQL command to make an existing user an admin:

    # Replace 'your-email@example.com' with your actual email
npm run db:studio

# Then in Drizzle Studio:
# 1. Go to the 'users' table
# 2. Find your user by email
# 3. Change the 'role' field to 'admin'
# 4. Save

    Or use psql directly (if you have DATABASE_URL configured):

    UPDATE users 
SET role = 'admin' 
WHERE email = 'your-email@example.com';

    üîç Troubleshooting Login Issues

    Issue: "Invalid credentials"

    Possible Causes: 1. Wrong email or password 2. User doesn't exist in database 3. Password doesn't meet minimum requirements

    Solutions: 1. Double-check your email and password 2. Try registering if you haven't already 3. Ensure password is at least 8 characters

    Issue: "Registration is restricted"

    Cause: You're trying to register but you're not the first user and don't have an invitation.

    Solution: - If you need admin access, use Option 1 or 2 above - Or ask an existing admin to invite you

    Issue: Database connection error

    Cause: Database not running or DATABASE_URL not configured

    Solution: 

    # Check if .env file exists
ls -la .env

# Verify DATABASE_URL is set
cat .env | grep DATABASE_URL

# Run migrations
npm run db:migrate

    üìä Check Current Users

    To see what users exist in your database:

    # Option 1: Use Drizzle Studio (Recommended)
npm run db:studio
# Then open http://localhost:4983 and browse the 'users' table

# Option 2: Use psql (if DATABASE_URL is configured)
# Add to your shell profile:
# export DATABASE_URL="your-database-url"

    üéØ Default Admin Credentials (Development)

    For development only, you can create a default admin user:

    Using Drizzle Studio:

    1. Run npm run db:studio
    2. Open http://localhost:4983
    3. Go to the 'users' table
    4. Click "Add Row"
    5. Fill in:
    6. name: Admin User
    7. email: admin@localbridge.dev
    8. role: admin
    9. passwordHash: (see below for how to generate)

    Generate Password Hash:

    # Create a quick script to hash a password
node -e "const bcrypt = require('bcryptjs'); bcrypt.hash('admin123', 10).then(hash => console.log(hash));"

    Then use the output as the passwordHash value.

    üîí Security Best Practices

    For Development:

    • ‚úÖ Use simple credentials (e.g., admin@localbridge.dev / admin123)
    • ‚úÖ Don't worry about password strength
    • ‚úÖ Can reset database anytime

    For Production:

    • ‚ùå Never use default credentials
    • ‚úÖ Use strong, unique passwords (16+ characters)
    • ‚úÖ Enable 2FA if available
    • ‚úÖ Use environment-specific admin emails
    • ‚úÖ Rotate credentials regularly

    üìù Quick Start Commands

    # 1. Ensure database is set up
npm run db:migrate

# 2. Start the application
npm run dev

# 3. Navigate to registration
# Open http://localhost:8081/register

# 4. Register as first user (becomes admin automatically)
# Name: Admin User
# Email: admin@localbridge.dev
# Password: admin123 (or your choice)

# 5. Login with your credentials
# Go to http://localhost:8081/login

    üéì Understanding Roles

    Available Roles:

    • admin: Full access to everything
    • editor: Can create and edit content
    • viewer: Read-only access
    • user: Standard user access

    Admin Capabilities:

    • ‚úÖ Manage all users
    • ‚úÖ Send invitations
    • ‚úÖ View all projects
    • ‚úÖ Access analytics
    • ‚úÖ Manage organizations
    • ‚úÖ Delete any resource

    üöÄ Next Steps After Login

    Once you're logged in as admin:

    1. Create an organization (optional)
    2. Click the organization switcher

    3. Select "Create Organization"

    4. Invite team members

    5. Go to user management

    6. Send invitations with appropriate roles

    7. Create your first project

    8. Navigate to Projects

    9. Click "Create Project"

    10. Generate a prototype

    11. Open a project
    12. Use the prompt templates
    13. Generate AI-powered code

    üìû Still Having Issues?

    Check These:

    1. ‚úÖ Is the dev server running? (npm run dev)
    2. ‚úÖ Is the database migrated? (npm run db:migrate)
    3. ‚úÖ Can you access http://localhost:8081?
    4. ‚úÖ Check browser console for errors (F12)
    5. ‚úÖ Check server logs in terminal

    Common Errors:

    "Failed to fetch" - Backend not running - Wrong API URL - CORS issue

    "Database error" - Database not migrated - DATABASE_URL incorrect - PostgreSQL not running

    "Invalid token" - Token expired (7 days) - JWT_SECRET changed - Clear localStorage and login again
