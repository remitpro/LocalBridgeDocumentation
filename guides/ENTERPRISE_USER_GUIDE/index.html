    LocalBridge Enterprise User Guide: Assistants, Agents, and Tools

    Welcome to the LocalBridge Enterprise User Guide. This document provides a comprehensive walkthrough for creating and managing AI Assistants, specialized Agents, and Custom Tools within the LocalBridge platform.

    These features enable you to build a powerful, autonomous AI workforce tailored to your specific enterprise needsâ€”running entirely locally and securely.

    1. Core Concepts

    Before diving into creation, it's important to understand the hierarchy of components:

    • Assistant: The primary interface for the user. Think of it as a "Department Head" or "Team Lead". It holds the main context, conversation history, and persona. It delegates tasks to Agents.
    • Agent: A specialized worker with a specific role (e.g., " Researcher", "Python Coder", "Legal Analyst"). Agents operate under an Assistant and have specific instructions and autonomy levels.
    • Tool: A specific capability given to an Agent (e.g., "Web Search", "Database Query", "Send Email"). Tools are defined by a function name and a JSON schema describing their parameters.

    2. Managing Assistants

    Assistants are the entry point for all interactions.

    Creating a New Assistant

    1. Open the LocalBridge application.
    2. Click the Assistant Selector dropdown in the top header (defaults to "New Chat" or current assistant name).
    3. Select "Create New" or click the "+" icon in the sidebar.
    4. Fill in the General Settings:
      • Name: distinct name (e.g., "Engineering Lead").
      • Description: Purpose of this assistant.
      • Model: Select a local LLM (e.g., llama3.1:latest).
      • Category: Grouping (Coding, Writing, etc.).
      • System Prompt: The core personality and rules.
        • Enterprise Tip: Be specific about compliance and tone. "You are a helpful assistant" is too vague. Use: "You are a Senior DevOps Engineer. You prioritize security and idempotency in all shell scripts."

    Advanced Parameters

    • Temperature: Controls creativity. Low (0-40) for code/facts; High (70-100) for creative writing.
    • Public/Private: Toggle "Public" to share this assistant template with other users on your local instance (if multi-user is enabled).

    3. Specialized Agents

    Agents are where the real work happens. They allow you to break down complex workflows.

    Adding an Agent to an Assistant

    1. In the Assistant Editor, switch to the "Specialized Agents" tab.
    2. Click "Add Agent".
    3. Configure the Agent:
      • Name: e.g., "Code Reviewer".
      • Role: Select from presets (Researcher, Writer, Coder, etc.) or define a custom one.
      • Instructions: Extremely detailed step-by-step logic for this specific agent.
        • Example: "Review the provided code for SQL injection vulnerabilities. Start by identifying user inputs. Then check if parameterized queries are used."
      • Autonomy Level:
        • Low: Agent must ask for user approval before every tool execution.
        • Medium: Agent can execute safe tools but asks for critical ones.
        • High: Agent runs fully autonomously until the task is done.
      • Priority: (0-100) Higher priority agents are consulted first by the Assistant.

    4. Custom Tools

    Tools provide Agents with the ability to interact with the outside world or perform specific computations.

    Defining a Tool

    In the Agent creation dialog, scroll to the "Tools & Capabilities" section.

    1. Click "Add Tool".
    2. Function Name: The exact function name the AI will call (e.g., lookup_customer_order, generate_report). Use snake_case.
    3. Description: A clear natural language description of what the tool does. The AI uses this to decide when to use the tool.

    4. Parameters (JSON Schema): Define the inputs strictly using JSON Schema. This ensures the AI provides structured, valid data.

      Example: Customer Lookup Tool

      {
  "type": "object",
  "properties": {
    "customerId": {
      "type": "string",
      "description": "The unique ID of the customer (e.g. CUST-123)"
    },
    "includeHistory": {
      "type": "boolean",
      "description": "Whether to fetch order history"
    }
  },
  "required": ["customerId"]
}

    Best Practices for Tools

    • Descriptive Names: search_knowledge_base is better than search.
    • Explicit Descriptions: "Searches the internal Q4 2025 financial documentation for specific keywords."
    • Simple Parameters: Keep schemas flat if possible. Complex nested objects can confuse smaller models.

    5. Enterprise Workflows

    Example: Automated Code Audit System

    Assistant: "Tech Lead" * System Prompt: "You oversee a team of developers. Your goal is to ensure code quality standards."

    Agent 1: "Security Auditor" * Role: Reviewer * Instructions: "Scan code for OWASP Top 10 vulnerabilities." * Tools: run_static_analysis, check_cve_database

    Agent 2: "Refactoring Bot" * Role: Coder * Instructions: "Take the findings from the Auditor and propose code changes." * Tools: apply_git_patch, run_unit_tests

    Workflow: 1. User enters: "Audit this file." 2. Tech Lead analyzes request and delegates to Security Auditor. 3. Security Auditor uses run_static_analysis, finds an issue, reports back. 4. Tech Lead delegates to Refactoring Bot to fix it. 5. Refactoring Bot generates code, runs unit_tests. 6. Tech Lead presents the final fixed code to the User.

    6. Troubleshooting

    • Agent not using tools: Check if the "Description" clearly matches the user's intent. The AI matches intent to description.
    • Invalid Parameter Errors: Verify your JSON Schema in the Tool Manager. Ensure required fields are marked as "required".
    • Looping: If an agent gets stuck in a loop, try lowering its Autonomy Level or refining its Instructions to include explicit "stopping conditions".
