    üîê Admin Login - Quick Fix

    Problem

    Cannot login with admin credentials.

    ‚úÖ Solution (Choose One)

    Option 1: Register as First User (Easiest)

    If no users exist yet:

    1. Go to http://localhost:8081/register
    2. Fill in the form:
    3. Name: Admin User
    4. Email: admin@localbridge.dev
    5. Password: admin123 (or your choice)
    6. Click "Register"
    7. You're now logged in as admin!

    Note: The first user to register automatically becomes an admin.

    Option 2: Use the Create Admin Script

    If users already exist:

    # Run the interactive script
npx tsx scripts/create-admin.js

# Follow the prompts:
# - Enter admin name (or press Enter for default)
# - Enter admin email (or press Enter for default)
# - Enter admin password (or press Enter for default)

# Default credentials:
# Email: admin@localbridge.dev
# Password: admin123

    Option 3: Use Drizzle Studio (Visual)

    # 1. Open Drizzle Studio
npm run db:studio

# 2. Navigate to http://localhost:4983

# 3. Go to 'users' table

# 4. Either:
#    a) Find your user and change 'role' to 'admin'
#    b) Add a new user with role 'admin'

# 5. For password hash, use:
node -e "const bcrypt = require('bcryptjs'); bcrypt.hash('admin123', 10).then(hash => console.log(hash));"

    Option 4: Reset Database (Development Only)

    # WARNING: Deletes all data!
npm run db:reset
npm run db:migrate

# Then register as first user (becomes admin)

    üéØ Recommended Approach

    For Development: Use Option 1 or Option 2 - Quick and easy!

    For Production: Use Option 3 - More control and security

    üìù Default Admin Credentials

    After using Option 1 or 2:

    Email: admin@localbridge.dev
Password: admin123

    Change these in production!

    ‚úÖ Verify It Works

    1. Go to http://localhost:8081/login
    2. Enter your admin credentials
    3. Click "Sign In"
    4. You should be redirected to the chat page
    5. Check navigation - you should see all menu items

    üö® Still Not Working?

    Check These:

    # 1. Is the server running?
npm run dev

# 2. Is the database migrated?
npm run db:migrate

# 3. Check server logs for errors
# Look in the terminal where you ran 'npm run dev'

# 4. Check browser console
# Press F12 and look for errors

    Common Issues:

    "Invalid credentials" - Wrong email/password - User doesn't exist - Try Option 2 to create/update user

    "Failed to fetch" - Backend not running - Check http://localhost:3001 is accessible

    "Database error" - Run npm run db:migrate - Check DATABASE_URL in .env

    üìû Need More Help?

    See the full guide: ADMIN_LOGIN_GUIDE.md

    Quick Commands: 

    # Create admin user
npx tsx scripts/create-admin.js

# Open database viewer
npm run db:studio

# Reset database (dev only)
npm run db:reset && npm run db:migrate

# Start application
npm run dev
