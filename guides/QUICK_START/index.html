    LocalBridge Quick Start Guide

    Prerequisites

    1. Node.js: v20.15.0 or higher
    2. PostgreSQL: Database instance (local or cloud)
    3. Ollama: Local LLM server (optional for AI features)

    Installation

    # Clone repository
git clone <repository-url>
cd chat-companion

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env
# Edit .env with your configuration

    Environment Setup

    Create a .env file in the root directory:

    # Database
DATABASE_URL=postgresql://user:password@localhost:5432/localbridge

# Authentication
JWT_SECRET=your-super-secret-jwt-key-change-this
ENCRYPTION_KEY=your-32-character-encryption-key-min

# Ollama (AI)
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=llama3.1:latest

# OAuth (Optional)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GOOGLE_CALLBACK_URL=http://localhost:3001/api/auth/google/callback

# Server
PORT=3001
NODE_ENV=development

    Database Setup

    # Run migrations
npm run db:migrate

# (Optional) Open Drizzle Studio to view database
npm run db:studio

    Running the Application

    Development Mode (Recommended)

    # Starts both frontend and backend with hot reload
npm run dev
    • Frontend: http://localhost:5173
    • Backend API: http://localhost:3001
    • Drizzle Studio: http://localhost:4983 (if running)

    Production Mode

    # Build the application
npm run build

# Start production server
npm start

    Ollama Setup (For AI Features)

    1. Install Ollama: 

      # macOS
brew install ollama

# Or download from https://ollama.ai

    2. Start Ollama: 

      ollama serve

    3. Pull a model: 

      ollama pull llama3.1:latest

    First-Time Usage

    1. Access the application: Navigate to http://localhost:5173
    2. Register an account: Click "Register" and create your account
    3. Create an organization (optional): Use the organization switcher in the navigation
    4. Create a project: Go to "Projects" and create your first project
    5. Generate a prototype: Enter a prompt and let AI generate code for you

    Common Commands

    # Development
npm run dev              # Start dev servers
npm run dev:client       # Start only frontend
npm run dev:server       # Start only backend

# Building
npm run build            # Build for production
npm run build:client     # Build frontend only
npm run build:server     # Build backend only

# Database
npm run db:generate      # Generate migration files
npm run db:migrate       # Apply migrations
npm run db:push          # Push schema (dev only)
npm run db:studio        # Open Drizzle Studio
npm run db:reset         # Reset database (WARNING)

# Code Quality
npm run lint             # Run ESLint
npm run lint:fix         # Fix ESLint errors
npm run type-check       # TypeScript type checking
npm run format           # Format code with Prettier

# Testing
npm test                 # Run tests
npm run test:watch       # Run tests in watch mode
npm run test:coverage    # Generate coverage report

    Troubleshooting

    Database Connection Issues

    # Verify DATABASE_URL is correct
echo $DATABASE_URL

# Test connection with psql
psql $DATABASE_URL

    Ollama Not Responding

    # Check if Ollama is running
curl http://localhost:11434/api/tags

# Restart Ollama
ollama serve

    Port Already in Use

    # Change PORT in .env file
PORT=3002

# Or kill the process using the port
lsof -ti:3001 | xargs kill -9

    Build Errors

    # Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install

# Clear build cache
rm -rf dist dist-server .vite
npm run build

 src/                    # Frontend source
    components/         # React components
    pages/             # Page components
    context/           # React contexts
    hooks/             # Custom hooks
    lib/               # Utilities
 server/                # Backend source
    routes/            # API routes
    lib/               # Server utilities
    middleware/        # Express middleware
    db/                # Database config
    __tests__/         # Backend tests
 public/                # Static assets
 dist/                  # Build output

    Default Credentials

    For development, you can create an admin user:

    -- Run this in your PostgreSQL database
INSERT INTO users (id, email, name, password, role)
VALUES (
  gen_random_uuid(),
  'admin@localbridge.dev',
  'Admin User',
  '$2a$10$...',  -- bcrypt hash of 'admin123'
  'admin'
);

    Or register normally through the UI.
