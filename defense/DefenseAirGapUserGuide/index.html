    Introduction

    The Defense Air-Gapped Operations module provides enterprise-grade security tools for classified environments, SCIF facilities, and air-gapped networks. This system ensures complete network isolation while providing essential security operations including:

    • AES-256 Encryption for files and text
    • Audit Logging for all security events
    • Session Management with recording and playback
    • Network Isolation Verification
    • Secure File Transfer with integrity checking
    • Terminal Operations for system diagnostics

    [!IMPORTANT] This module is designed for classified operations. All actions are logged and monitored for security compliance.

    Getting Started

    Prerequisites

    • Access to Enterprise Solutions Hub
    • Appropriate clearance level
    • Understanding of air-gapped security protocols

    Accessing the Module

    1. Navigate to Enterprise Solutions from the main menu
    2. Click on the Defense tab (marked with red "Classified" badge)
    3. Review the security notice and proceed

    Activating Air-Gap Protocol

    Step-by-Step Activation

    1. Locate the Air-Gap Protocol Card
    2. Find the card labeled "Air-Gap Protocol" on the Defense page

    3. Current status will show "OPEN configuration" when inactive

    4. Click Activate Protocol

    5. Press the red "Activate Protocol" button

    6. System will immediately enter isolated mode

    7. Verify Activation

    8. Status changes to "ISOLATED from public networks"
    9. AIR-GAPPED badge appears in the header

    10. Security dashboard becomes accessible

    11. Automatic Session Start

    12. A new security session is automatically created
    13. All actions will be logged to this session

    [!WARNING] Once activated, the system will attempt to block all external network connections. Ensure all necessary files are available locally before activation.

    Deactivation

    To deactivate air-gap mode: 1. Click the "Deactivate Protocol" button 2. Current session will be automatically ended 3. System returns to OPEN configuration

    Security Dashboard

    The Security Dashboard provides real-time monitoring of your air-gapped environment.

    Dashboard Components

    Status Overview

    • Security State: OPEN, ISOLATED, or LOCKED
    • Clearance Level: Current user clearance (UNCLASSIFIED TOP_SECRET)
    • Network Status: ISOLATED or CONNECTED
    • Integrity Score: System integrity percentage (target: 100%)

    System Metrics

    • Uptime: Time since air-gap activation
    • Total Sessions: Number of security sessions created
    • Active Sessions: Currently running sessions
    • Audit Events: Total security events logged

    Current Session

    • Session ID: Unique identifier for tracking
    • Started: Session start time
    • Events: Number of events in current session
    • Status: Active or Inactive

    Recent Audit Events

    View the last 5 security events with: - Event description - Event type (MODE_CHANGE, FILE_TRANSFER, ENCRYPTION, etc.) - Timestamp - Severity level (low, medium, high, critical)

    Emergency Actions

    • Emergency Lockdown: Immediately locks the system and terminates all sessions
    • Use only in case of security breach

    Exporting Audit Logs

    1. Click "Export Logs" button on the dashboard
    2. Logs are downloaded as JSON file
    3. File includes all events with timestamps and metadata

    Secure File Transfer

    Transfer files securely with optional encryption and integrity verification.

    Uploading Files

    1. Navigate to File Transfer Tab

    2. Click "File Transfer" in the sub-navigation

    3. Configure Encryption (Optional)

    4. Check "Enable AES-256 Encryption"
    5. Enter a strong password

    6. Toggle "Show password" if needed

    7. Select Files

    8. Click the upload area or drag and drop files
    9. Multiple files can be selected

    10. Each file is processed individually

    11. Monitor Progress

    12. Files appear in the transfer queue
    13. Progress bar shows encryption/transfer status

    14. SHA-256 hash is calculated for integrity

    15. Transfer Complete

    16. Green checkmark indicates success
    17. Download button becomes available

    Downloading Files

    1. Locate File in Queue

    2. Find the completed transfer

    3. Click Download Button

    4. If encrypted, file is automatically decrypted using the password

    5. If not encrypted, original file is downloaded

    6. Verify Integrity

    7. Compare SHA-256 hash if needed
    8. Hash is displayed under each file

    Transfer Queue Management

    • Remove Transfer: Click trash icon to remove from queue
    • Clear Completed: Remove all completed transfers at once

    [!TIP] Always verify file integrity using the SHA-256 hash, especially for critical files.

    Encryption & Decryption

    Encrypt and decrypt text or files using AES-256-GCM encryption.

    Encrypting Text

    1. Navigate to Encryption Tab

    2. Click "Encryption" in the sub-navigation

    3. Set Password

    4. Enter a strong encryption password

    5. Or click the key icon to generate a random key

    6. Enter Plaintext

    7. Type or paste text to encrypt in the text area

    8. Click "Encrypt Text"

    9. System generates encrypted ciphertext

    10. Salt and IV (Initialization Vector) are also generated

    11. Save Encrypted Data

    12. Copy the ciphertext, salt, and IV
    13. Or click "Export" to download as JSON file

    [!IMPORTANT] Save all three values: ciphertext, salt, and IV. All are required for decryption.

    Decrypting Text

    1. Switch to Decrypt Tab

    2. Click the "Decrypt" tab

    3. Enter Encrypted Data

    4. Paste ciphertext
    5. Paste salt

    6. Paste IV

    7. Enter Password

    8. Must be the same password used for encryption

    9. Click "Decrypt Text"

    10. Decrypted text appears below
    11. If password is incorrect, error message is shown

    Random Key Generation

    • Click the key icon next to password field
    • System generates a cryptographically secure random key
    • Key is automatically set as the password

    Secure Terminal

    Execute commands in a secure, air-gapped terminal environment.

    Available Commands

    Command Description Usage
    help Show all available commands help
    status Check system security status status
    scan Perform local network scan scan
    encrypt Encrypt a string encrypt <text>
    audit View recent audit log entries audit
    network Network diagnostics network
    session View current session info session
    integrity Run system integrity check integrity
    metrics Display security metrics metrics
    export Export audit logs export
    clear Clear terminal history clear
    lock Emergency lockdown lock

    Threat Intelligence

    Monitor global threat signals and security alerts in real-time.

    Features

    • DEFCON Level: Current defense readiness condition
    • SIGINT Status: Signals intelligence monitoring status
    • System Uptime: Network uptime percentage
    • Active Nodes: Visual grid of monitoring nodes
    • Live Signal Feed: Real-time intercepted signals

    Signal Types

    • Intercept (Blue): Communication intercepts
    • Threat (Red): Identified threats
    • Anomaly (Yellow): Unusual patterns

    Signal Information

    Each signal displays: - Source (SAT-4, GND-STATION-Lagos, INTEL-BUJA, DRONE-RECON) - Geographic coordinates - Severity level - Timestamp

    [!NOTE] Threat Intelligence requires active air-gap mode to function.

    Best Practices

    Security

    1. Always Verify Air-Gap Status
    2. Check for AIR-GAPPED badge in header

    3. Verify network status shows "ISOLATED"

    4. Use Strong Passwords

    5. Minimum 16 characters
    6. Mix of letters, numbers, and symbols

    7. Consider using generated keys

    8. Verify File Integrity

    9. Always check SHA-256 hashes

    10. Compare before and after transfer

    11. Regular Integrity Checks

    12. Run integrity command periodically
    13. Monitor audit logs for anomalies

    Operational

    1. Session Management
    2. Start new session for each work period
    3. Export session logs before ending

    4. Review audit events regularly

    5. File Transfers

    6. Encrypt sensitive files
    7. Keep transfer queue organized

    8. Clear completed transfers

    9. Documentation

    10. Export audit logs regularly
    11. Maintain offline backups
    12. Document all security incidents

    Troubleshooting

    Air-Gap Protocol Won't Activate

    Problem: Button doesn't respond or activation fails

    Solutions: 1. Refresh the page 2. Check browser console for errors 3. Verify you have appropriate clearance 4. Try logging out and back in

    File Transfer Fails

    Problem: File upload or encryption fails

    Solutions: 1. Check file size (large files may take longer) 2. Verify password is set if encryption is enabled 3. Try with a smaller file first 4. Check browser console for errors

    Decryption Fails

    Problem: "Decryption failed" error message

    Solutions: 1. Verify password is correct 2. Ensure salt and IV are correctly copied 3. Check for extra spaces in pasted values 4. Verify ciphertext is complete

    Terminal Commands Not Working

    Problem: Commands return errors or don't execute

    Solutions: 1. Type help to see available commands 2. Check command syntax 3. Ensure air-gap mode is active 4. Try clear and retry command

    Network Alerts in Air-Gap Mode

    Problem: "Network connection detected" alerts

    Solutions: 1. Disconnect physical network cable 2. Disable Wi-Fi 3. Check for VPN connections 4. Use network command to diagnose

    FAQ

    Q: What happens to my data when I activate air-gap mode?

    A: All data remains local. The system attempts to block external network connections and all operations are performed offline.

    Q: Can I transfer files between air-gapped systems?

    A: Yes, use the Secure File Transfer feature to encrypt files, then transfer via approved physical media (USB drives, etc.).

    Q: How long are audit logs kept?

    A: The system keeps the last 1000 audit events in memory. Export logs regularly for permanent storage.

    Q: What encryption algorithm is used?

    A: AES-256-GCM (Galois/Counter Mode) with PBKDF2 key derivation (100,000 iterations).

    Q: Can I use this on a mobile device?

    A: The interface is responsive and works on tablets. Phone screens may be too small for optimal use.

    Q: What should I do if I forget my encryption password?

    A: Encrypted data cannot be recovered without the password. Always store passwords securely.

    Q: How do I know if my system is truly air-gapped?

    A: Use the network command in the terminal to verify isolation. The dashboard also shows network status.

    Q: Can multiple users access air-gap mode simultaneously?

    A: Each user has their own session. Sessions are isolated and independently managed.

    Q: What's the difference between ISOLATED and LOCKED states?

    A: ISOLATED is normal air-gap mode. LOCKED is emergency mode that terminates all sessions and requires administrator intervention.

    Q: How do I export my session data?

    A: Use the "Export Logs" button on the dashboard or the export command in the terminal.

    Support

    For technical support or security incidents: - Contact your system administrator - Review audit logs for detailed error information - Export logs before reporting issues
