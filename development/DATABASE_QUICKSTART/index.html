    PostgreSQL Database Integration - Quick Start

    ‚úÖ What's Been Set Up

    Your LocalBridge application now has full PostgreSQL database integration! Here's what's ready:

    Backend Server

    • ‚úÖ Express.js API server
    • ‚úÖ Drizzle ORM for type-safe database operations
    • ‚úÖ REST API endpoints for conversations and messages
    • ‚úÖ Security middleware (CORS, Helmet, Rate Limiting)
    • ‚úÖ Input validation with Zod

    Database Schema

    • ‚úÖ Users table (for multi-user support)
    • ‚úÖ Conversations table (chat sessions)
    • ‚úÖ Messages table (individual messages)
    • ‚úÖ Proper indexes for performance
    • ‚úÖ Foreign key constraints for data integrity

    Migration Files

    • ‚úÖ Generated migration: drizzle/0000_fuzzy_paibok.sql
    • ‚úÖ Migration script ready to run

    üöÄ Next Steps

    1. Add Database URL to .env

    Your .env file needs the database connection string. Since it's protected by .gitignore (for security), you need to add it manually:

    # Open your .env file and add this line:
DATABASE_URL=postgresql://neondb_owner:npg_DwYLAyv65hWg@ep-cold-dust-abvtnhea-pooler.eu-west-2.aws.neon.tech/neondb?sslmode=require&channel_binding=require

    Or use the setup script:

    chmod +x setup-database.sh
./setup-database.sh

    2. Run Database Migrations

    Once the DATABASE_URL is in your .env file:

    npm run db:migrate

    This will create the tables in your PostgreSQL database.

    3. Start the Application

    # Start both frontend and backend
npm run dev

# Or run separately:
npm run dev:client  # Frontend (port 8080)
npm run dev:server  # Backend (port 3001)

    üìä What You Can Do Now

    View Your Database

    npm run db:studio

    Opens Drizzle Studio at https://local.drizzle.studio to browse your data.

    API Endpoints Available

    Conversations: - GET /api/conversations - List all conversations - GET /api/conversations/:id - Get conversation with messages - POST /api/conversations - Create new conversation - PATCH /api/conversations/:id - Update conversation - DELETE /api/conversations/:id - Delete conversation

    Messages: - GET /api/messages/:conversationId - Get messages - POST /api/messages - Create message - DELETE /api/messages/:id - Delete message

    Test the API

    # Health check
curl http://localhost:3001/health

# Create a conversation
curl -X POST http://localhost:3001/api/conversations \
  -H "Content-Type: application/json" \
  -d '{"title":"Test Chat","model":"llama3.1:latest"}'

# List conversations
curl http://localhost:3001/api/conversations

    üéØ Migration from localStorage

    The app previously used localStorage. To migrate:

    1. Export your data (optional): 

      // In browser console
const data = localStorage.getItem('ollama-chat-conversations');
console.log(JSON.parse(data));

    2. Use the new database - conversations will now persist server-side

    3. Clear localStorage (optional): 

      localStorage.removeItem('ollama-chat-conversations');

    ‚ùì Troubleshooting

    "DATABASE_URL not set" error

    Solution: Add the connection string to your .env file (see step 1 above)

    "Cannot connect to database"

    Solution: 1. Check your internet connection 2. Verify the DATABASE_URL is correct 3. Ensure Neon database is active

    CORS errors

    Solution: 1. Verify backend is running on port 3001 2. Check CORS_ORIGIN in .env matches frontend URL 3. Ensure VITE_API_URL is set correctly
