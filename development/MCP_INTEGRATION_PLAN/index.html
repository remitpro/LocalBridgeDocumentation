    MCP Integration Plan

    Overview

    The Model Context Protocol (MCP) allows applications to provide context (resources, prompts, tools) to LLMs. Integrating an MCP server into localbridge will allow external agents (like Claude Desktop or other MCP clients) to interact with the application's data and agents.

    Proposed Strategy

    1. Architecture

    We will implement an MCP Server that runs alongside the existing Express backend. - Transport: Server-Sent Events (SSE) over HTTP is the best fit for a web application backend. We will expose: - GET /api/mcp/sse: For initializing the connection. - POST /api/mcp/messages: For handling client messages.

    2. Capabilities to Expose

    Resources (Data Access)

    • Conversations: readable as resources conversation://{id}.
    • Documents: readable as resources document://{id}.
    • Project Files: readable as resources.

    Tools (Action Execution)

    • list_agents: Returns a list of available agents in the system.
    • execute_agent: Allows an external LLM to trigger a localbridge agent to perform a task.
    • create_task: Create a new task in the system.

    Prompts

    • system_prompt: Expose the core system prompt used by the application.

    3. Implementation Steps

    1. Dependencies: Install @modelcontextprotocol/sdk.
    2. Server Implementation: Create server/mcp/server.ts to define the MCP server, capabilities, and request handlers.
    3. Express Integration: Mount the MCP server on the existing Express app in server/index.ts.
    4. Testing: Verify connection using an MCP client inspector or a simple script.

    Benefits

    • Interoperability: Connects localbridge to the broader ecosystem of AI tools.
    • Extensibility: Allows other specialized agents to "hire" localbridge agents for subtasks.
    • Context Sharing: Seamlessly share the application's knowledge base with external models.
