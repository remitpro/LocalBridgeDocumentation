    Overview

    Base URL

    Development: http://localhost:3001/api
Production: https://your-domain.com/api

    API Version

    Current version: v1

    Content Type

    All requests and responses use application/json unless otherwise specified.

    Authentication

    Most endpoints require authentication via JWT token in the Authorization header.

    Authentication

    Headers

    Authorization: Bearer <jwt_token>
Content-Type: application/json

    Getting a Token

    Obtain a JWT token by logging in via the /api/auth/login endpoint.

    Auth Endpoints

    POST /api/auth/login

    Authenticate a user and receive a JWT token.

    Request: 

    POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "securepassword"
}

    Response (200 OK): 

    {
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "user_123",
    "email": "user@example.com",
    "name": "John Doe",
    "role": "editor"
  }
}

    Errors: - 400 - Missing email or password - 401 - Invalid credentials - 429 - Too many login attempts

    POST /api/auth/logout

    Logout the current user (client-side token removal).

    Request: 

    POST /api/auth/logout
Authorization: Bearer <token>

    Response (200 OK): 

    {
  "message": "Logged out successfully"
}

    GET /api/auth/me

    Get current user information.

    Request: 

    GET /api/auth/me
Authorization: Bearer <token>

    Response (200 OK): 

    {
  "id": "user_123",
  "email": "user@example.com",
  "name": "John Doe",
  "role": "editor",
  "createdAt": "2026-01-01T00:00:00.000Z",
  "lastLoginAt": "2026-01-04T18:00:00.000Z"
}

    Errors: - 401 - Unauthorized (invalid or missing token)

    GET /api/auth/google

    Initiate Google OAuth flow.

    Request: 

    GET /api/auth/google

    Response: Redirects to Google OAuth consent screen.

    GET /api/auth/google/callback

    Google OAuth callback (handled automatically).

    GET /api/auth/microsoft

    Initiate Microsoft OAuth flow.

    Request: 

    GET /api/auth/microsoft

    Response: Redirects to Microsoft OAuth consent screen.

    GET /api/auth/microsoft/callback

    Microsoft OAuth callback (handled automatically).

    Projects Endpoints

    GET /api/projects

    Get all projects for the current user.

    Request: 

    GET /api/projects
Authorization: Bearer <token>

    Query Parameters: - page (optional): Page number (default: 1) - limit (optional): Items per page (default: 20, max: 100) - sort (optional): Sort field (default: createdAt) - order (optional): Sort order (asc/desc, default: desc)

    Response (200 OK): 

    {
  "projects": [
    {
      "id": "proj_123",
      "name": "My Project",
      "description": "Project description",
      "githubRepo": "https://github.com/user/repo",
      "createdAt": "2026-01-01T00:00:00.000Z",
      "updatedAt": "2026-01-04T18:00:00.000Z",
      "prototypeCount": 5
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 42,
    "pages": 3
  }
}

    GET /api/projects/:id

    Get a specific project by ID.

    Request: 

    GET /api/projects/proj_123
Authorization: Bearer <token>

    Response (200 OK): 

    {
  "id": "proj_123",
  "name": "My Project",
  "description": "Project description",
  "githubRepo": "https://github.com/user/repo",
  "createdAt": "2026-01-01T00:00:00.000Z",
  "updatedAt": "2026-01-04T18:00:00.000Z",
  "prototypes": [
    {
      "id": "proto_456",
      "name": "User Card",
      "description": "User profile card component",
      "code": "import React from 'react'...",
      "createdAt": "2026-01-02T00:00:00.000Z",
      "versionNumber": 1
    }
  ]
}

    Errors: - 404 - Project not found - 403 - Access denied

    POST /api/projects

    Create a new project.

    Request: 

    POST /api/projects
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "New Project",
  "description": "Optional description",
  "githubRepo": "https://github.com/user/repo"
}

    Response (201 Created): 

    {
  "id": "proj_789",
  "name": "New Project",
  "description": "Optional description",
  "githubRepo": "https://github.com/user/repo",
  "createdAt": "2026-01-04T18:00:00.000Z",
  "updatedAt": "2026-01-04T18:00:00.000Z"
}

    Errors: - 400 - Invalid input (missing name, invalid GitHub URL) - 401 - Unauthorized

    PUT /api/projects/:id

    Update a project.

    Request: 

    PUT /api/projects/proj_123
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "Updated Project Name",
  "description": "Updated description",
  "githubRepo": "https://github.com/user/new-repo"
}

    Response (200 OK): 

    {
  "id": "proj_123",
  "name": "Updated Project Name",
  "description": "Updated description",
  "githubRepo": "https://github.com/user/new-repo",
  "updatedAt": "2026-01-04T18:05:00.000Z"
}

    Errors: - 404 - Project not found - 403 - Access denied - 400 - Invalid input

    DELETE /api/projects/:id

    Delete a project and all its prototypes.

    Request: 

    DELETE /api/projects/proj_123
Authorization: Bearer <token>

    Response (200 OK): 

    {
  "message": "Project deleted successfully",
  "deletedPrototypes": 5
}

    Errors: - 404 - Project not found - 403 - Access denied

    Prototypes Endpoints

    POST /api/generate/prototype

    Generate a new prototype using AI.

    Request: 

    POST /api/generate/prototype
Authorization: Bearer <token>
Content-Type: application/json

{
  "projectId": "proj_123",
  "prompt": "Create a user profile card with avatar and buttons",
  "name": "User Profile Card",
  "model": "llama3.1:latest"
}

    Response (200 OK - Server-Sent Events): 

    data: {"chunk": "import React from 'react';\n"}
data: {"chunk": "import { Card } from '@/components/ui/card';\n"}
...
data: {"done": true, "prototypeId": "proto_456", "versionNumber": 1}

    SSE Event Types: - chunk - Code chunk - done - Generation complete - error - Generation error - correlationId - Request tracking ID

    Errors: - 400 - Invalid input (missing prompt, invalid model) - 401 - Unauthorized - 403 - Access denied (project not owned) - 429 - Rate limit exceeded - 504 - Generation timeout (2 minutes)

    GET /api/prototypes/:id

    Get all users (admin only).

    Request: 

    GET /api/users
Authorization: Bearer <admin_token>

    Query Parameters: - page (optional): Page number - limit (optional): Items per page - role (optional): Filter by role

    Response (200 OK): 

    {
  "users": [
    {
      "id": "user_123",
      "email": "user@example.com",
      "name": "John Doe",
      "role": "editor",
      "createdAt": "2026-01-01T00:00:00.000Z",
      "lastLoginAt": "2026-01-04T18:00:00.000Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 50,
    "pages": 3
  }
}

    Errors: - 403 - Forbidden (not admin)

Authorization: Bearer <admin_token>
Content-Type: application/json

{
  "email": "newuser@example.com",
  "password": "securepassword",
  "name": "Jane Smith",
  "role": "editor"

    Error Handling

    Error Response Format

    {
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": {
    "field": "Additional context"
  },
  "correlationId": "req_abc123"
}

    HTTP Status Codes

    Code Meaning
    200 Success
    201 Created
    400 Bad Request
    401 Unauthorized
    403 Forbidden
    404 Not Found
    409 Conflict
    429 Too Many Requests
    500 Internal Server Error
    504 Gateway Timeout

    Common Error Codes

    Code Description
    INVALID_INPUT Request validation failed
    UNAUTHORIZED Authentication required
    FORBIDDEN Insufficient permissions
    NOT_FOUND Resource not found
    RATE_LIMIT_EXCEEDED Too many requests
    GENERATION_TIMEOUT AI generation timed out
    GENERATION_FAILED AI generation error

    Rate Limiting

    Limits

    • Authentication: 5 requests per minute
    • Generation: 10 requests per 15 minutes
    • Other endpoints: 100 requests per minute

    Headers

    X-RateLimit-Limit: 10
X-RateLimit-Remaining: 7
X-RateLimit-Reset: 1704398400

    Rate Limit Response

    {
  "error": "Rate limit exceeded",
  "code": "RATE_LIMIT_EXCEEDED",
  "retryAfter": 300
}

    Examples

    Complete Flow Example

    // 1. Login
const loginResponse = await fetch('http://localhost:3001/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'password123'
  })
});

const { token, user } = await loginResponse.json();

// 2. Create Project
const projectResponse = await fetch('http://localhost:3001/api/projects', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    name: 'My Project',
    description: 'Test project'
  })
});

const project = await projectResponse.json();

// 3. Generate Prototype
const generateResponse = await fetch('http://localhost:3001/api/generate/prototype', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    projectId: project.id,
    prompt: 'Create a button component',
    model: 'llama3.1:latest'
  })
});

// Handle SSE stream
const reader = generateResponse.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');

  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      if (data.chunk) {
        console.log('Code:', data.chunk);
      }
      if (data.done) {
        console.log('Prototype ID:', data.prototypeId);
      }
    }
  }
}
