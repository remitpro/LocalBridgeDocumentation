    Quick Start: Assistants & Agents

    Run This Now!

    Step 1: Run Migration

    npm run db:migrate

    This creates 3 new tables: - assistants - Custom AI personas - agents - Specialized workers - agent_executions - Task tracking

    Step 2: Start the Server

    npm run dev

    Test the API

    Create an Assistant

    curl -X POST http://localhost:3001/api/assistants \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Code Reviewer",
    "description": "Expert code reviewer",
    "systemPrompt": "You are an expert code reviewer focusing on quality, security, and best practices.",
    "model": "llama3.1:latest",
    "temperature": 30,
    "category": "coding",
    "tags": ["code", "review", "quality"]
  }'

    Create an Agent

    curl -X POST http://localhost:3001/api/agents \
  -H "Content-Type: application/json" \
  -d '{
    "assistantId": "ASSISTANT_ID_FROM_ABOVE",
    "name": "Security Analyzer",
    "role": "analyst",
    "description": "Analyzes code for security vulnerabilities",
    "instructions": "Review code for common security issues like SQL injection, XSS, authentication flaws, etc.",
    "priority": 100,
    "autonomyLevel": "high"
  }'

    List Assistants

    curl http://localhost:3001/api/assistants

    List Agents

    curl http://localhost:3001/api/agents

    API Endpoints

    Assistants

    • GET /api/assistants - List all
    • GET /api/assistants/:id - Get one
    • POST /api/assistants - Create
    • PATCH /api/assistants/:id - Update
    • DELETE /api/assistants/:id - Delete
    • POST /api/assistants/:id/duplicate - Duplicate

    Agents

    • GET /api/agents - List all
    • GET /api/agents/:id - Get one
    • POST /api/agents - Create
    • PATCH /api/agents/:id - Update
    • DELETE /api/agents/:id - Delete
    • POST /api/agents/:id/execute - Execute task
    • PATCH /api/agents/:id/toggle - Enable/disable

    Example Use Cases

    1. Code Review Team

    {
  "name": "Code Review Team",
  "agents": [
    { "role": "analyst", "name": "Security Checker" },
    { "role": "analyst", "name": "Performance Optimizer" },
    { "role": "reviewer", "name": "Best Practices" }
  ]
}

    2. Content Creation

    {
  "name": "Content Team",
  "agents": [
    { "role": "researcher", "name": "Topic Researcher" },
    { "role": "writer", "name": "Content Writer" },
    { "role": "reviewer", "name": "Editor" }
  ]
}

    3. Data Analysis

    {
  "name": "Data Analyst",
  "agents": [
    { "role": "researcher", "name": "Data Collector" },
    { "role": "analyst", "name": "Statistical Analyzer" },
    { "role": "writer", "name": "Report Generator" }
  ]
}

    Agent Roles

    • researcher - Gathers information
    • writer - Creates content
    • analyst - Analyzes data
    • coder - Writes/reviews code
    • reviewer - Provides feedback
    • planner - Creates plans
    • executor - Executes tasks
    • coordinator - Manages agents

    Configuration Tips

    Temperature Settings

    • 0-30: Consistent, factual (good for code review)
    • 40-70: Balanced (good for general chat)
    • 80-100: Creative (good for content creation)

    Priority Levels

    • 100+: Critical, execute first
    • 50-99: Normal priority
    • 0-49: Low priority

    Autonomy Levels

    • low: Requires approval
    • medium: Supervised execution
    • high: Fully autonomous
