    Production Deployment Guide

    This guide covers deploying the LocalBridge application to production environments.

    Prerequisites

    • Production server with Docker support (recommended)
    • Domain name with DNS configured
    • SSL/TLS certificate (Let's Encrypt recommended)
    • Ollama instance accessible from your deployment

    Environment Setup

    1. Create Production Environment File

    cp .env.production .env.production.local

    2. Configure Production Variables

    Edit .env.production.local:

    # Ollama Configuration
VITE_OLLAMA_HOST=https://your-ollama-host.com
VITE_OLLAMA_MODEL=llama3.1:latest

# Analytics (Optional)
VITE_POSTHOG_API_KEY=your_production_key
VITE_POSTHOG_HOST=https://app.posthog.com

# Environment
VITE_APP_ENV=production
VITE_APP_VERSION=1.0.0

# Feature Flags
VITE_ENABLE_ANALYTICS=true
VITE_ENABLE_ERROR_REPORTING=true

    Deployment Options

    Docker Deployment

    Option 1: Docker Compose (Recommended)

    1. Update docker-compose.yml with your configuration

    2. Deploy: 

      docker-compose up -d

    3. Verify: 

      docker-compose ps
docker-compose logs -f chat-companion

    Option 2: Docker with Reverse Proxy

    1. Build the image: 

      docker build -t chat-companion:v1.0.0 .

    2. Run with Nginx reverse proxy: 

      docker run -d \
  --name chat-companion \
  --restart unless-stopped \
  -p 8080:80 \
  --env-file .env.production.local \
  chat-companion:v1.0.0

    3. Configure Nginx reverse proxy (on host): 

      server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

    Kubernetes Deployment

    1. Create namespace: 

      kubectl create namespace chat-companion

    2. Create secrets: 

      kubectl create secret generic chat-companion-env \
  --from-env-file=.env.production.local \
  -n chat-companion

    3. Apply Kubernetes manifests: 

      # deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: chat-companion
  namespace: chat-companion
spec:
  replicas: 3
  selector:
    matchLabels:
      app: chat-companion
  template:
    metadata:
      labels:
        app: chat-companion
    spec:
      containers:
      - name: chat-companion
        image: chat-companion:v1.0.0
        ports:
        - containerPort: 80
        envFrom:
        - secretRef:
            name: chat-companion-env
        resources:
          requests:
            memory: "128Mi"
            cpu: "100m"
          limits:
            memory: "256Mi"
            cpu: "200m"
        livenessProbe:
          httpGet:
            path: /health
            port: 80
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 5
---
apiVersion: v1
kind: Service
metadata:
  name: chat-companion
  namespace: chat-companion
spec:
  selector:
    app: chat-companion
  ports:
  - port: 80
    targetPort: 80
  type: LoadBalancer


    kubectl apply -f deployment.yaml

    Traditional Server Deployment

    1. Build the application: 

      npm run build

    2. Install Nginx: 

      sudo apt update
sudo apt install nginx

    3. Copy build files: 

      sudo cp -r dist/* /var/www/chat-companion/

    4. Configure Nginx: 

      sudo cp nginx.conf /etc/nginx/sites-available/chat-companion
sudo ln -s /etc/nginx/sites-available/chat-companion /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

    5. Setup SSL with Let's Encrypt: 

      sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com

    Cloud Platforms

    AWS (Elastic Beanstalk)

    1. Install EB CLI: 

      pip install awsebcli

    2. Initialize and deploy: 

      eb init -p docker chat-companion
eb create production-env
eb deploy

    Google Cloud Platform (Cloud Run)

    1. Build and push image: 

      gcloud builds submit --tag gcr.io/PROJECT_ID/chat-companion

    2. Deploy: 

      gcloud run deploy chat-companion \
  --image gcr.io/PROJECT_ID/chat-companion \
  --platform managed \
  --region us-central1 \
  --allow-unauthenticated

    Azure (Container Instances)

    1. Create resource group: 

      az group create --name chat-companion-rg --location eastus

    2. Deploy container: 

      az container create \
  --resource-group chat-companion-rg \
  --name chat-companion \
  --image chat-companion:latest \
  --dns-name-label chat-companion \
  --ports 80

    Vercel

    1. Install Vercel CLI: 

      npm i -g vercel

    2. Deploy: 

      vercel --prod

    Netlify

    1. Install Netlify CLI: 

      npm i -g netlify-cli

    2. Deploy: 

      netlify deploy --prod --dir=dist

    Post-Deployment

    1. Verify Deployment

    # Check health endpoint
curl https://your-domain.com/health

# Check application
curl -I https://your-domain.com

    2. Configure DNS

    Point your domain to the deployment: - A Record: Point to server IP - CNAME: Point to cloud platform URL

    3. Enable HTTPS

    Uncomment HSTS header in nginx.conf: 

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;

    4. Setup Monitoring

    • Configure uptime monitoring (UptimeRobot, Pingdom)
    • Setup error tracking (Sentry)
    • Enable analytics (PostHog, Google Analytics)

    Monitoring

    Health Checks

    The application exposes a /health endpoint:

    curl https://your-domain.com/health
# Response: healthy

    Logs

    Docker: 

    docker logs -f chat-companion
docker-compose logs -f

    Kubernetes: 

    kubectl logs -f deployment/chat-companion -n chat-companion

    Nginx: 

    sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log

    Metrics

    Monitor these key metrics: - Response time - Error rate - CPU/Memory usage - Request rate - Bundle size

    Troubleshooting

    Container won't start

    # Check logs
docker logs chat-companion

# Verify environment variables
docker exec chat-companion env

# Check health
docker exec chat-companion wget -O- http://localhost/health

    502 Bad Gateway

    • Verify container is running
    • Check Nginx configuration
    • Verify port mappings
    • Check firewall rules

    SSL Certificate Issues

    # Renew Let's Encrypt certificate
sudo certbot renew

# Test renewal
sudo certbot renew --dry-run

    High Memory Usage

    • Reduce Docker memory limits
    • Enable swap
    • Scale horizontally with load balancer

    CORS Issues

    Update nginx.conf to add CORS headers: 

    add_header Access-Control-Allow-Origin "https://your-domain.com" always;
add_header Access-Control-Allow-Methods "GET, POST, OPTIONS" always;
add_header Access-Control-Allow-Headers "Authorization, Content-Type" always;

    Rollback

    Docker

    # List images
docker images

# Rollback to previous version
docker stop chat-companion
docker rm chat-companion
docker run -d --name chat-companion chat-companion:v0.9.0

    Kubernetes

    # Rollback deployment
kubectl rollout undo deployment/chat-companion -n chat-companion

# Check rollout status
kubectl rollout status deployment/chat-companion -n chat-companion

    Security Checklist

    • [ ] HTTPS enabled with valid certificate
    • [ ] HSTS header enabled
    • [ ] CSP headers configured
    • [ ] Environment variables secured
    • [ ] Regular security updates scheduled
    • [ ] Firewall rules configured
    • [ ] Rate limiting enabled
    • [ ] Backup strategy implemented
    • [ ] Monitoring and alerting configured
    • [ ] Access logs enabled

    Performance Optimization

    1. Enable CDN for static assets
    2. Configure caching headers properly
    3. Enable HTTP/2 in Nginx
    4. Use compression (Gzip/Brotli)
    5. Optimize images and assets
    6. Monitor bundle size regularly
