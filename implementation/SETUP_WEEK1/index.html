    Quick Setup Guide - Week 1 Fixes

    Prerequisites

    • Node.js v20.15.0 or higher
    • PostgreSQL database
    • Ollama running locally

    Step 1: Install Dependencies

    cd /Users/teejay/dev/chat-companion
npm install

    Step 2: Generate Encryption Key

    Choose one method:

    Option A: Using OpenSSL (Recommended)

    openssl rand -base64 32

    Option B: Using Node.js

    node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

    Option C: Using Python

    python3 -c "import secrets; print(secrets.token_urlsafe(32))"

    Step 3: Update Environment Variables

    Add to your .env file:

    # CRITICAL: Required for GitHub token encryption
ENCRYPTION_KEY=<paste-your-generated-key-here>

# Verify these existing variables:
DATABASE_URL=postgresql://user:password@localhost:5432/localbridge
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=llama3.1:latest
JWT_SECRET=your-jwt-secret
PORT=3001
NODE_ENV=development

    Step 4: Test Encryption

    Create a test file test-encryption.ts:

    import { encrypt, decrypt, validateEncryptionKey } from './server/lib/crypto.js';

// Test encryption key
if (validateEncryptionKey()) {
    console.log(' Encryption key is valid');

    // Test encryption/decryption
    const testData = 'test-github-token';
    const encrypted = encrypt(testData);
    const decrypted = decrypt(encrypted);

    if (testData === decrypted) {
        console.log(' Encryption/decryption working correctly');
    } else {
        console.error(' Encryption/decryption failed');
    }
} else {
    console.error(' Invalid encryption key');
}

    Run test: 

    npx tsx test-encryption.ts

    Step 5: Start Server

    npm run dev

    Step 6: Verify Health Check

    curl http://localhost:3001/health

    Expected response: 

    {
    "healthy": true,
    "timestamp": "2026-01-04T...",
    "checks": {
        "database": {
            "healthy": true,
            "responseTime": 15
        },
        "ollama": {
            "healthy": true,
            "responseTime": 120
        }
    }
}

    Step 7: Test Rate Limiting

    Try making multiple requests quickly:

    # Should succeed (first 10 requests)
for i in {1..10}; do
    curl -X POST http://localhost:3001/api/generate/prototype \
        -H "Content-Type: application/json" \
        -H "Authorization: Bearer YOUR_TOKEN" \
        -d '{"projectId":"...","prompt":"test"}'
done

# Should fail with 429 (11th request)
curl -X POST http://localhost:3001/api/generate/prototype \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -d '{"projectId":"...","prompt":"test"}'

    Troubleshooting

    Error: "ENCRYPTION_KEY environment variable is required"

    • Make sure .env file has ENCRYPTION_KEY set
    • Restart the server after adding the key

    Error: "ENCRYPTION_KEY must be at least 32 characters"

    • Generate a longer key using one of the methods above
    • Recommended: 32-64 characters

    Error: "Failed to decrypt data"

    • Existing tokens in database are not encrypted
    • Need to run migration (see below)

    Health Check Returns 503

    • Check if PostgreSQL is running
    • Check if Ollama is running on port 11434
    • Check database connection string

    Migrating Existing Data

    If you have existing projects with GitHub tokens:

    Option A: Manual Update (Small Dataset)

    1. Export existing tokens
    2. Re-add projects with new tokens (will be encrypted automatically)

    Option B: Migration Script (Large Dataset)

    Create migrate-encrypt-tokens.ts:

    import { db } from './server/db/index.js';
import { projects } from './server/db/schema.js';
import { encrypt } from './server/lib/crypto.js';
import { eq, isNotNull } from 'drizzle-orm';

async function migrateTokens() {
    console.log('Starting token encryption migration...');

    // Get all projects with tokens
    const projectsWithTokens = await db.query.projects.findMany({
        where: isNotNull(projects.githubToken)
    });

    console.log(`Found ${projectsWithTokens.length} projects with tokens`);

    for (const project of projectsWithTokens) {
        try {
            // Try to decrypt - if it fails, it's plaintext
            try {
                decrypt(project.githubToken!);
                console.log(` Project ${project.id} already encrypted`);
                continue;
            } catch {
                // Token is plaintext, encrypt it
                const encrypted = encrypt(project.githubToken!);

                await db.update(projects)
                    .set({ githubToken: encrypted })
                    .where(eq(projects.id, project.id));

                console.log(` Encrypted token for project ${project.id}`);
            }
        } catch (error) {
            console.error(` Failed to encrypt project ${project.id}:`, error);
        }
    }

    console.log('Migration complete!');
}

migrateTokens().catch(console.error);

    Run migration: 

    npx tsx migrate-encrypt-tokens.ts

    Security Checklist

    • [ ] ENCRYPTION_KEY is set and at least 32 characters
    • [ ] ENCRYPTION_KEY is different from JWT_SECRET
    • [ ] .env file is in .gitignore
    • [ ] Encryption test passes
    • [ ] Health check returns 200
    • [ ] Rate limiting is working
    • [ ] Existing tokens are encrypted (if applicable)
